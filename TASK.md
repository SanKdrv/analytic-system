Есть распределенная RAG система.
На одном сервере backend, RAG и БД.
На втором сервере ollama LLM.
На третьем сервере языковая модель для векторизации и вставки в векторную БД.

Твоя задача - сделать систему мониторинга и аналитики над RAG системой.
Нужно иметь фронтенд (VueJS с докеризацией), который будет показывать grafana графики с ресурсами трех серверов.
Также нужно иметь возможность поменять параметры RAG бэкенда (Потом скорректирую). Бэкенд перезагрузится автоматически с новыми параметрами через эндпоинт.
Также нужно иметь возможность в реальном времени просматривать пропускную способность и качество RAG ответов.
Для этого нужно постоянно передавать в backend тестовые данные, и брать результат с бэкенда RAG.

Backend аналитической системы напиши на Python. Параметры, конфиги и креды серверов должны быть вынесены в .env файл. Сделай часть функциями заглушками, но главное - чтобы работала grafana и показывала ресурсы трёх серверов.
